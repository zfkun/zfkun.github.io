ComfyUI 合理规划与瘦身重装

2025-11-25
5分钟阅读时长
2066字
阅读
tech , linux , windows , ai

最近 Qwen-Image-Edit-2509 比较火热, 甚至 FLUX2 也已发布 (GGUF量化也同步跟进), 于是乎重新打开吃灰已久的秋叶启动器, 经过重复着下载,安装,报错,修复,再安装,再调试后, 算是上手体验了一段时间, 的确挺不错的.

有意思的是, 发布文章之时 Z-Image 也刚发布了 Z-Image-Turbo, 卡着 FLUX2 的发布后发布, 真是有点意思 ~~

但是看着岌岌可危的硬盘, 还是不得不要执行拖延了一年多的计划了: 清理磁盘, 重装comfyui

如今都 2025 (其实, 也可以说 2026) 了, 很多AI刚流行爆发时的一些模型,早已不再那么好用(或者说不如现在更新的后来者好用), 是时候重新来过, 只保留主流较近的模型, 对于是不是玩玩的话是足够的.

合理规划

有时为了测试不同模型, 需要安装各自不同的一堆插件,依赖,甚至c源码构建 等等.

如果, 总是在一个 comgfyui 环境里这么来回折腾, 早晚会碰到依赖冲突 或 版本错乱 的问题,

比如, 一个插件要求 transformer < 1.0, 而另一个插件要求 tranformer > 1.0, 阁下该如何应对呢 ?

所以, 最优的方案应该是, 本机部署 N 份 ComgfyUI, 每一份仅共享模型文件, 不共享插件, 只要合理划分好用途, 稳定性肯定要比 长时间在同一个 ComgfyUI 里折腾要高得多.

看似复杂, 实则简单, 用空间换时间.

主要是 多份 python_embeded 占用的空间较多

现在已经不像 SDXL 之前的轻量时代了, 现在大模型动不动都要几十G

建议 ui 和 models 放在 固态硬盘 (模型加载耗时短的多), workspace 放 机械硬盘 (数据安全第一)

安装步骤

上面啰嗦了一堆, 其实并不重要, 不喜欢可以不理会无妨.

进入正题:

下载

直接下载 官方便携版 即可

这2年换回 windows 为主了, mac 暂时吃灰中

windows 环境, 玩AI大都用的 秋叶启动器, 不过随着时代发展进步, 官方也推出了客户端, 对国内环境友好的模型站也出现了不少(比如 魔搭社区), 显卡硬件迭代(比如 N卡50系也已都用上) 等等

重装 comfyui 的话, 已没有再使用 启动器 的必要了, 直接用 便携版 就可以了, 基本能做到最轻量, 有问题随能随时自行调试定位并解决

解压放置位置: D:\ai\ui\comfyui\comfyui_xxx

_xxx 只是举例, 实际命名根据情况修改

PIP 加速

打开 PowerShell 或 终端

# 设置镜像源 (清华大学)
D:\ai\ui\comfyui\comfyui_xxx\python_embeded\python.exe -m pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple

# 设置HTTP信任域名 (如果用的 http:// 地址的话)
D:\ai\ui\comfyui\comfyui_xxx\python_embeded\python.exe -m pip config set global.trusted-host pypi.tuna.tsinghua.edu.cn

执行成功后, 配置会写入文件: %APPDATA%\pip\pip.ini

所以, 也可以手工直接修改此文件 (如果没有, 新建即可)

[global]
index-url = https://pypi.tuna.tsinghua.edu.cn/simple
trusted-host = pypi.tuna.tsinghua.edu.cn

可以通过命令行确认当前配置状态

D:\ai\ui\comfyui\comfyui_xxx\python_embeded\python.exe -m pip config list -v

比如, 输出内容:

For variant 'global', will try loading 'C:\ProgramData\pip\pip.ini'
For variant 'user', will try loading 'C:\Users\Administrator\pip\pip.ini'
For variant 'user', will try loading 'C:\Users\Administrator\AppData\Roaming\pip\pip.ini'
For variant 'site', will try loading 'D:\ai\ComfyUI_windows_portable\python_embeded\pip.ini'
global.index-url='https://pypi.tuna.tsinghua.edu.cn/simple'
global.trusted-host='pypi.tuna.tsinghua.edu.cn'

安装 ComfyUI-Manager

ComfyUI-Manager 目前还没有被官方集成到 便携版 里, 所以还是需要手工下载安装一下

本机有 [git环境], 则:


# 进入到 ComfyUI　的 custom_nodes 目录下
cd D:\ai\ui\comfyui\comfyui_xxx\ComfyUI\custom_nodes

# 下载插件 (可以访问 github)
git clone https://github.com/ltdrdata/ComfyUI-Manager comfyui-manager

# 下载插件 (不可以访问 github)
# git clone https://ghfast.top/https://github.com/ltdrdata/ComfyUI-Manager comfyui-manager

本机没有 git环境, 则:

直接下载 zip (可以访问 github)

直接下载 zip (不可以访问 github)

然后, 解压到 D:\ai\ui\comfyui\comfyui_xxx\ComfyUI\custom_nodes 内

注意, 目录层级, 有时候 winrar 之类的解压缩工具, 会解压后自动多套一层文件夹

迁移配置

如果希望重装的 ComfyUI 集成原来的配置 (已经存在的 model, clip, encoder, vae 等等),

那么就修改下启动脚本的参数, 指定相关参数即可 (如果要 多份环境隔离, 配置目录也做隔离即可)

# 修改 `run_nvidia_gpu.bat` 追加自定义配置 (另外2个bat也一样修改)
.\python_embeded\python.exe -s ComfyUI\main.py --windows-standalone-build --extra-model-paths-config E:/ai/workspace/comfyui/extra_model_paths.yaml --user-directory E:/ai/workspace/comfyui/user --input-directory E:/ai/inputs/comfyui --output-directory E:/ai/outputs/comfyui

启动 ComfyUI

到此, 一个干净的基础 ComfyUI 就算可以了.

直接运行 run_nvidia_gpu.bat 或 (run_cpu.bat, run_nvidia_gpu_fast_fp16_accumulation.bat) 即可

其实, 科学上网还是需要有的, 不然启动 comgfyui 阶段, 还是依旧存在一些请求无法正常造成各种异常失败

这里暂时先不做那么细了, 毕竟是打算快速重装复用, 细节优化的等空了再单独写一写.

起手插件

起手而已, 并不需要大而全, 只需小而精且实用够用即可, 再好的东西偶尔用按需随装随用才比较合适

以下列举了一些完成空白全新 comfyui 出装后, 起手的插件, 按需选择后就够用了其实

通用

  • ComfyUI-nunchaku 双截棍名不虚传

    nunchaku 的插件安装, 也经常会碰到问题, 常见的就是安装依赖后引起的, 各种依赖包的版本冲突问题

    比如 numpy 的版本 与 opencv-python 的依赖版本冲突 就比较常见

    　以目前 1.1.0 的轮子 和 插件 来说, numpy==1.26.4 应该就可以解决问题

  • ComfyUI-GGUF GGUF量化低显福音

高清放大

  • ComfyUI-SeedVR2_VideoUpscaler

抠图识别

  • ComfyUI-RMBG

    这个插件安装估计会碰到不少小问题, 个人的经验建议: 这个插件装在独立的一份 comfyui 实例中比较好, 否则可能会时不时的因依赖问题造成崩溃反复重装

    比如: 新版对 sam3 做了支持, 但是安装最新插件后, 重装依赖过程会波折不断

    • 报错 缺少 triton 依赖, 需手工安装 triton-windows 而不是 triton

      # 卸载旧版本 (如果有安装过的话)
D:\ai\ui\comfyui\comfyui_xxx\python_embeded\python.exe -m pip uninstall triton

# 安装新版本
D:\ai\ui\comfyui\comfyui_xxx\python_embeded\python.exe -m pip install -U "triton-windows<3.6"

      Triton 版本与 PyTorch 的版本对应关系如下: 官方说明

      PyTorch Triton
      2.4 3.1
      2.5 3.1
      2.6 3.2
      2.7 3.3
      2.8 3.4
      2.9 3.5

      我用的都是最新的 PyTorch 2.9.1, 所以我直接装的 3.5 的: "triton-windows<3.6"

      假如, 你的是 PyTorch 2.8 那么就需要改下参数安装 3.4 的: "triton-windows<3.5"

    • 报错 缺少 decord 依赖, 需手工安装 decord 依赖

      D:\ai\ui\comfyui\comfyui_xxx\python_embeded\python.exe -m pip install decord

    • 报错 缺少 ftfy 依赖, 需手工安装 "ftfy==6.1.1"

      D:\ai\ui\comfyui\comfyui_xxx\python_embeded\python.exe -m pip install "ftfy==6.1.1"

    • 遇到 UnicodeDecodeError: 'gbk' codec ... 错误, 需手工下载 GroundingDINO 项目源码修改编译

      # 解压后, 修改文件 `setup.py` 的 `README.md` 的读取代码部分, 增加 `encoding="utf-8"` 参数
with open('README.md', encoding="utf-8") as readme_file:
    readme = readme_file.read()

      # 然后, 在源码目录下, 执行手工安装
D:\ai\ui\comfyui\comfyui_xxx\python_embeded\python.exe -m pip install .
